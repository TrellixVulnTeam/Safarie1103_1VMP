The :target hack

The :target pseudo-selector targets the element with an ID matching the part of the URL following an octothorpe (#). In other words, like the :hover CSS rule is applied when the state of the element is being hovered over, the :target CSS rule is applied when the state of the URL has the matching ID at its end.

 

Exercise 5b - Click Events (part 2)

In this exercise you'll recreate the zoom in/out feature above.

  1. Open the completed 5-css-click-events.html.
  2. Add an <h1> containing the name of this exercise.
  3. Add a <figure> element, with a nested <figcaption> element.
  4. In the <figcaption>, write "zoom in  |  zoom out"
    NOTE: the pipe character (|) is on the key above the RETURN key.
  5. Insert an image in the <figure> (but before the <figcaption>). Use a placeholder image from lorempixel.com:
    <img src="http://lorempixel.com/400/400" />
    NOTE: this requests a random 400x400px image. Do not assign a width or height.
  6. Assign this image an ID of "zoom".
  7. Make the "zoom in" text a link that leads to "#zoom" and the "zoom out" text a link that leads to "#".
  8. Write a CSS rule for the #zoom element that sets a 150px width, and a transition of all properties over 1 sec. with the ease-in-out easing function.
  9. Write a CSS rule using the #zoom:target selector that assigns a width of 400px.
  10. Save the page and test it in a browser.
    TIP: if the page is long enough to scroll, zooming out will scroll back to the top. You can change the zoom out link from "#" to "#zoomout" and assign that ID to the figure element to avoid that. But as you see on this page, the shrinking size of the zoomed element scrolls the page in odd ways. This method is therefore only good for pages without much content that might force a scroll.
